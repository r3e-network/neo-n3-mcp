# Release 1.6.4 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Prepare and publish patch release `1.6.4` by updating all current-version references, refreshing release documentation, and creating the GitHub release after verification.

**Architecture:** Use `npm version patch --no-git-tag-version` as the canonical source for `package.json` and `package-lock.json`, then align repo-wide current-version references that describe the live release surface. Update release automation/docs to match the actual codebase layout (`src/version.ts` owns the server version now), verify builds/tests, then commit, push, tag, and publish the GitHub release.

**Tech Stack:** npm, Git, GitHub CLI, TypeScript, Jest, Markdown docs

**Files:**

- Modify: `package-lock.json`
