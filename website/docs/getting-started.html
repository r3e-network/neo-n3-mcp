1 Installation

Choose your preferred installation method to get started with Neo N3 MCP Server.

Install via NPM

The easiest way to get started. Requires Node.js 18+ and npm.

bash 
# Install globally
npm install -g @r3e/neo-n3-mcp

# Or install locally in your project
npm install @r3e/neo-n3-mcp
üí° Prerequisites

Make sure you have Node.js 18+ and npm installed on your system.

Install via Docker

Run Neo N3 MCP Server in a containerized environment.

bash 
# Pull the latest image
docker pull r3e/neo-n3-mcp:latest

# Run the container
docker run -p 3000:3000 \
  -e NEO_NETWORK=mainnet \
  r3e/neo-n3-mcp:latest
yaml 
# docker-compose.yml
version: '3.8'
services:
  neo-mcp:
    image: r3e/neo-n3-mcp:latest
    ports:
      - "3000:3000"
    environment:
      - NEO_NETWORK=mainnet
      - LOG_LEVEL=info
    restart: unless-stopped

Install for Claude Desktop

Integrate directly with Claude Desktop for AI-powered blockchain interactions.

json 
{
  "mcpServers": {
    "neo-n3": {
      "command": "npx",
      "args": ["@r3e/neo-n3-mcp"],
      "env": {
        "NEO_NETWORK": "mainnet"
      }
    }
  }
}

2 Configuration

Configure your Neo N3 MCP Server for optimal performance and security.

Environment Variables

bash 
# Network Configuration
NEO_NETWORK=mainnet           # or 'testnet'
NEO_RPC_URL=https://mainnet1.neo.coz.io:443

# Security Settings
ENABLE_CONFIRMATION=true      # Require confirmation for write operations
MAX_REQUEST_SIZE=1048576     # Maximum request size in bytes

# Logging
LOG_LEVEL=info               # debug, info, warn, error
LOG_FORMAT=json              # json or text

# Performance
CACHE_TTL=300               # Cache time-to-live in seconds
MAX_CONCURRENT_REQUESTS=10   # Maximum concurrent requests

Network Selection

üåê Mainnet
Production network with real NEO and GAS tokens. Use for live applications.
NEO_NETWORK=mainnet
üß™ Testnet
Test network with free tokens. Perfect for development and testing.
NEO_NETWORK=testnet
‚ö†Ô∏è Security Best Practices

Always keep ENABLE_CONFIRMATION=true in production to prevent accidental transactions. Never expose private keys in environment variables.

3 First Steps

Let's verify your installation and run your first blockchain queries.

1. Start the Server

bash 
# If installed globally
neo-n3-mcp

# If installed locally
npx @r3e/neo-n3-mcp

# With custom configuration
NEO_NETWORK=testnet neo-n3-mcp

2. Verify Connection

bash 
# Test the connection with a simple blockchain info request
curl -X POST http://localhost:3000/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/call",
    "params": {
      "name": "get_blockchain_info"
    },
    "id": 1
  }'

3. Get Network Status

javascript 
// Example response for get_blockchain_info
{
  "jsonrpc": "2.0",
  "result": {
    "content": [
      {
        "type": "text",
        "text": "Neo N3 Blockchain Information:\n- Network: mainnet\n- Block Height: 12,345,678\n- Validators: 7\n- Status: Online"
      }
    ]
  },
  "id": 1
}
‚úÖ Success!

If you received a response similar to above, your Neo N3 MCP Server is running correctly and connected to the blockchain network.

4. Explore Available Tools

bash 
# List all available tools
curl -X POST http://localhost:3000/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "tools/list",
    "id": 1
  }'
üéØ Quick Test Commands

Try these commands to verify different features:

  • get_network_mode - Check current network
  • get_block_count - Get latest block height
  • list_famous_contracts - See available contracts
